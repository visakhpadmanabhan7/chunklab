NimbusSync — Product Specification

NimbusSync is a file-synchronization service that keeps a folder consistent across many devices with end-to-end encryption and conflict-free merging.

Goals

  • Sub-second propagation of changes on a local network.
  • End-to-end encryption; the server never sees plaintext.
  • Deterministic conflict resolution with no data loss.

Non-goals

  • Real-time collaborative editing of a single document.
  • Version control semantics (branches, merges by hand).

Architecture

Each client maintains a Merkle tree of the synced folder. On change, the client diffs the tree, encrypts the changed blocks, and uploads them. Peers pull the manifest, fetch missing blocks, and apply them atomically. Conflicts are resolved by a last-writer-wins rule on a per-block basis, with the losing version preserved as a side file.

Security

Keys are derived from the user's passphrase with Argon2id. Block contents are encrypted with XChaCha20-Poly1305. The server stores only ciphertext and manifests and cannot read file names or contents.
